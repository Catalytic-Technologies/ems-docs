# Notifications

<span className="role-badge role-badge--admin">School Admin</span> <span className="role-badge role-badge--bursar">Bursar</span> <span className="role-badge role-badge--teacher">Teacher</span>

Notifications are short, automated alerts generated by the system when key events occur. They appear in the notifications panel and may also be sent by email or SMS.

## Viewing Notifications

Click the **bell icon** in the top navigation bar to see your recent notifications. Click **View All** to open the full notifications list.

## Types of Notifications

| Type | Triggered When |
|------|---------------|
| **Payment received** | A fee payment is recorded for a student |
| **Invoice generated** | A new invoice is created for a student |
| **Overdue fee** | A student's fee is past its due date |
| **Attendance alert** | A student is absent for N consecutive days |
| **Assessment due** | An assessment date is approaching |
| **Report pack ready** | A report pack is published for review |
| **New application** | A new admissions application is received |

## Notification Settings

Admins can configure which notifications are sent and via which channel (in-app, email, SMS) in **Settings → Notifications**.
